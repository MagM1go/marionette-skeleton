# И лирическое отступление
Но, чтобы не фанатеть сильно по определённым паттернам архитектуры, достаточно почитать про coupling и cohesion.

Вкратце:

**coupling** - чем меньше взаимозависимость между модулями, тем лучше.

**cohesion** - про сильную взаимосвязь компонентов модуля внутри самого себя (в Чистой архитектуре это, например, выражается в доменном слое)
